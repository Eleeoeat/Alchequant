# Alchequant GitHub 发布清单

## 推荐发布目录

建议在以下路径准备干净发布版：

```text
F:\Python Programming\Alchequant
```

当前开发目录保留为雏形和历史资料目录，发布目录只保留 GitHub 需要的内容。

## 应该包含

- `app.py`
- `src/`
- `scripts/download_data.py`
- `scripts/generate_report.py`
- `scripts/list_stocks.py`
- `notebooks/`
- `assets/`
- `.streamlit/config.toml`
- `data/stocks.db`
- `data/hs300_stocks.csv`
- `data/stock_list.txt`
- `requirements.txt`
- `README.md`
- `启动说明.md`
- `发布清单.md`
- `run.bat`
- `run_mac.sh`
- `.gitignore`

## 不应该包含

- `DeepSeek_api_Opencode.txt`
- `.env` 或任何 API Key 文件
- `.claude/`
- `.agents/`
- `.git/` 旧目录
- `__pycache__/`
- `scripts/import_csv.py`
- `scripts/create_notebooks.py`
- `scripts/create_stage3_notebooks.py`
- 大量历史 `results/charts/*.html`
- 大量历史 `results/reports/*.html`
- 临时压缩包和本机缓存

## GitHub 首次上传建议

在干净发布目录中执行：

```bash
git init
git add .
git commit -m "Initial release"
git branch -M main
```

然后创建 GitHub 仓库，并关联远程地址：

```bash
git remote add origin https://github.com/<your-user>/<repo>.git
git push -u origin main
```

也可以使用 GitHub CLI：

```bash
gh repo create Alchequant --public --source . --remote origin --push
```
